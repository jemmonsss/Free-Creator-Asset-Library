# Contributing to Free Creator Asset Library

Thank you for considering contributing! 🎉

This repository aims to build a **large library of creator-made CC0 assets** that anyone can use freely.

Before contributing, please read the rules below carefully.

---

# 📜 Contribution Requirements

All assets submitted to this repository must meet **ALL** of the following requirements:

### 1. Asset Ownership

Assets must be:

- **Created by you**, OR  
- Assets that you **legally own and have the right to redistribute**

You **must NOT upload** assets from:

- Asset marketplaces
- Paid asset packs
- Free asset packs that forbid redistribution
- Websites that only allow use inside projects

Many assets are **free to use but NOT free to redistribute**, which would violate their license if uploaded here.

---

### 2. License Requirement

All assets must be released under:

**CC0 (Creative Commons Zero)**

This means the asset becomes **public domain** and anyone can:

- Use it
- Modify it
- Redistribute it
- Use it commercially

Without attribution.

More info:  
https://creativecommons.org/publicdomain/zero/1.0/

---

### 3. Repository License vs Asset License

| Item | License |
|-----|------|
| Repository structure & documentation | MIT |
| Assets | CC0 |

Do **not apply MIT to assets**.  
Assets must be **CC0 only**.

---

# 📁 Folder Structure

Assets must be placed inside a folder named after your **GitHub username**.

Example:

```text
assets/
 ├── your-username/
 │   ├── 3d-assets/
 │   │   ├── models/
 │   │   ├── materials/
 │   │   └── scenes/
 │   │
 │   ├── 2d-assets/
 │   │   ├── sprites/
 │   │   ├── ui/
 │   │   └── textures/
 │   │
 │   ├── audio/
 │   │   ├── sound-effects/
 │   │   └── music/
 │   │
 │   └── misc/
```

This keeps each contributor's assets organized.

---

# 🧾 Asset Information

If possible, include a small `info.txt` or `README.md` inside your asset folder containing:

- Asset name
- Creator (your name / username)
- Creation tool (optional)
- File formats
- Any notes about usage

---

# ⚠️ Removal Policy

Assets may be removed if:

- They violate licensing rules
- They are reported as stolen
- They come from sources that forbid redistribution

If you repeatedly submit violating assets, you may be restricted from contributing.

---

# 🚀 How to Contribute

1. Fork the repository
2. Add your assets inside your username folder
3. Follow the folder structure
4. Create a Pull Request
5. Fill out the pull request template

Your assets will be reviewed before being merged.

---

Thank you for helping build a **truly free asset library for creators**!
